# economy-analyzer
Разработал на Python инструмент для анализа внутриигровой экономики: IQR‑выявление аномальных цен, LLM‑рекомендации, отчёт в Excel. Сократил анализ 1000+ предметов с 1,5 дней до 5 минут, доля пропущенных аномалий 3 %.

# Проблема
В игре с пользовательской торговлей (маркетплейс предметов) геймдизайнеры вручную проверяли цены на 1000+ предметов, чтобы найти «перекосы»: слишком дешёвые редкие вещи, подозрительно дорогие расходники и т. п. Это занимало 1–2 дня на каждый патч, и часть аномалий всё равно пропускали. Из‑за этого страдал баланс и экономика игры: игроки быстро находили «дыры» и фармили ресурсы.

Цель: сделать инструмент, который автоматически находит аномальные цены, объясняет, почему они подозрительны, и предлагает диапазон корректных цен. Результат должен быть понятен геймдизайнеру (не просто цифры, а выводы на человеческом языке).

# Что сделал
Написали скрипт, который загружает данные о продажах/ценах из CSV/базы, считает статистические метрики (среднее, медиана, IQR) и выделяет выбросы.
Для каждой аномальной строки сгенерировали понятное объяснение и рекомендации по цене с помощью LLM.
Вывели итоговый отчёт в Excel с колонками: предмет, текущая цена, рекомендуемый диапазон, причина аномалии, уровень риска.
Добавили CLI для запуска и тесты на граничные случаи (пустые данные, все одинаковые цены и т. д.).

# Как использовал ИИ
Генерация объяснений. Для каждой аномалии ИИ пишет короткое объяснение на русском в стиле «геймдизайнерского комментария»: «Цена слишком низкая для редкого предмета; вероятно, баг или ошибка выгрузки».
Формулирование рекомендаций. ИИ предлагает диапазон цен и обоснование: «Рекомендуемый диапазон: 1200–1800. Текущая цена 450 — ниже медианы по редкости “rare” на 73%».
Шаблоны и инструкции. С помощью ИИ подготовили шаблон входных данных, примеры отчётов и инструкцию для геймдизайнеров.
Черновики тестов. ИИ помог сгенерировать набор тестовых данных с разными типами аномалий (резкие пики, нулевые цены, дубликаты).

# Результаты
Время анализа 1200 предметов сократилось с ~1,5 дней до ~5 минут.
Доля пропущенных аномалий снизилась с ~20% до <3% (по итогам ручной проверки выборок).
Появился единый артефакт (Excel‑отчёт) с понятными выводами, который геймдизайнеры используют как основу для правок.
Инструмент интегрирован в пайплайн: отчёт формируется автоматически перед каждым балансным обновлением.


# Сложности и решения
Разные валюты и единицы. Решение: нормализация всех цен к одной базовой валюте и добавление колонки «currency_source».
Шум в данных. Решение: фильтрация дублей, удаление нулевых и отрицательных цен до анализа.
«Человеческий» вывод. Решение: чёткие промпты для ИИ с примерами и ограничениями по длине, чтобы текст был кратким и по делу.
